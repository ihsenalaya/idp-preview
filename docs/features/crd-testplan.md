# TestPlan CRD — Intelligent Test Selection

The **TestPlan** Custom Resource represents a decision about which test suites to run for a given Preview. It is either generated by the **test-strategist-agent** (AI-driven) or created manually, and is reviewed by the controller before test execution begins.

---

## What it's for

- Record the test-strategist agent's recommendation on which suites are mandatory (`mustRun`), advisable (`shouldRun`), or safe to skip (`canSkip`)
- Provide a grounding mechanism: the agent's decisions are captured as a CR and validated by the controller before tests run
- Enable transparent audit trails: every test decision is recorded with its confidence score and rationale
- Support manual override: developers can create TestPlans by hand and point the Preview to them

---

## What it does

When `testStrategy.mode: Auto` is set in a Preview:

1. **Controller creates TestPlan stub** in the Preview's namespace with `phase: Pending`
2. **Controller creates ephemeral Job** (curlimages/curl, TTL 5min) that calls the test-strategist-agent
3. **Agent reads** `spec.changeContext.changedFiles` + `spec.changeContext.diffPatch` from the Preview
4. **Agent writes** `spec.mustRun`, `spec.shouldRun`, `spec.canSkip` to the TestPlan
5. **Controller polls** for updates; once the agent finishes, the TestPlan is `phase: Ready`
6. **Controller validates** confidence ≥ `spec.testStrategy.confidenceThreshold` (default 70%)
7. **Controller accepts** the plan by setting `status.acceptedByController: true`
8. **Test suite runs** — only suites in `mustRun` + `shouldRun` are scheduled; `canSkip` suites are marked `Skipped`

## API Overview

```yaml
apiVersion: platform.company.io/v1alpha1
kind: TestPlan
metadata:
  name: pr-42-abc123
  namespace: preview-pr-42
spec:
  # Reference to the Preview that requested this plan
  previewRef:
    kind: Preview
    name: pr-42
    namespace: default

  # Who produced the plan
  generatedBy: Agent  # Agent | Manual | FallbackPolicy
  agentName: test-strategist-agent
  agentVersion: "1.0.5"

  # Agent's self-reported confidence (0-100)
  confidence: 75

  # Tests the agent considers mandatory
  mustRun:
    - suite: smoke
      name: "*"
      reason: "Always run baseline liveness tests."
    - suite: e2e
      name: "*"
      reason: "Frontend.py changed; E2E coverage required."
    - suite: regression
      name: "*"
      reason: "All changes require regression baseline."

  # Tests the agent recommends but does not mandate
  shouldRun:
    - suite: contract
      name: "*"
      reason: "API contract stability recommended for backend changes."

  # Tests the agent considers safe to skip
  canSkip:
    - suite: migration
      name: "*"
      reason: "No migration files changed."

  # Human-readable explanation (included in PR comment)
  rationale: |
    Frontend.py and app.py changed. Smoke and regression tests are baseline.
    E2E is mandatory because frontend UI may be affected. Contract tests
    are recommended for API stability but not critical for this diff.

  # Agent's estimate of total test time
  estimatedDurationSeconds: 120

  # Timestamps
  generatedAt: "2026-06-01T10:00:00Z"
  expiresAt: "2026-06-01T10:30:00Z"
  commitSHA: "abc123def456"

  # Propagated for correlation across related CRs
  correlationID: "reconcile-12345-67890"

status:
  # Lifecycle phase
  phase: Ready  # Pending | Ready | Stale | Rejected

  # Controller's acceptance decision
  acceptedByController: true
  acceptedAt: "2026-06-01T10:05:00Z"

  # Why the controller rejected this plan (if applicable)
  rejectionReason: ""
```

## Spec Fields

### References & Identity

| Field | Type | Purpose |
|-------|------|---------|
| `previewRef` | ObjectReference | The Preview CR that requested this plan |
| `correlationID` | string | Propagated from the Preview; ties all related CRs together |

### Agent Metadata

| Field | Type | Purpose |
|-------|------|---------|
| `generatedBy` | enum | Who produced this: `Agent` \| `Manual` \| `FallbackPolicy` |
| `agentName` | string | Agent identifier (e.g., `test-strategist-agent`) |
| `agentVersion` | string | Agent version (for debugging) |

### Decisions

| Field | Type | Purpose |
|-------|------|---------|
| `confidence` | int (0-100) | Agent's self-reported confidence in this plan |
| `mustRun[]` | []TestSelector | Tests the agent considers mandatory for this diff |
| `shouldRun[]` | []TestSelector | Tests the agent recommends but does not mandate |
| `canSkip[]` | []TestSelector | Tests the agent considers safe to skip |

### TestSelector structure

| Field | Type | Purpose |
|-------|------|---------|
| `suite` | string | Test category: `smoke` \| `contract` \| `regression` \| `migration` \| `e2e` \| `load` |
| `name` | string | Test identifier within the suite; use `"*"` to select all tests in the suite |
| `reason` | string | Human-readable explanation for why this test was selected or skipped |

### Metadata

| Field | Type | Purpose |
|-------|------|---------|
| `rationale` | string | Full explanation of the test plan decision, included verbatim in the PR comment |
| `estimatedDurationSeconds` | int | Agent's estimate of total test time |
| `generatedAt` | time | When the agent produced this plan |
| `expiresAt` | time | When this plan becomes invalid (typically commit SHA changes) |
| `commitSHA` | string | Head commit this plan was produced for |

---

## Status Fields

| Field | Type | Meaning |
|-------|------|---------|
| `phase` | string | Lifecycle: `Pending` (awaiting agent) → `Ready` (agent complete) → `Stale` (commit changed) → `Rejected` (failed validation) |
| `acceptedByController` | bool | Controller validated and accepted this plan (confidence ≥ threshold) |
| `acceptedAt` | time | When the controller accepted this plan |
| `rejectionReason` | string | Why the controller rejected this plan (e.g., "confidence 45% < threshold 70%") |

## Lifecycle

```
Preview.spec.testStrategy.mode = Auto
           │
           ▼
Controller creates TestPlan with phase=Pending
           │
           ▼
Controller creates ephemeral Job (curlimages/curl, TTL 5min)
           │
           ├─ Job calls test-strategist-agent A2A endpoint
           ├─ Agent reads Preview.spec.changeContext (diff, files, impacts)
           └─ Agent patches TestPlan.spec: mustRun[], shouldRun[], canSkip[]
           │
           ▼
Controller polls TestPlan status
           │
           ├─ phase=Pending?  ──► RequeueAfter=5s
           └─ phase=Ready?    ──► Validate confidence ≥ threshold
                                 ├─ YES ──► acceptedByController=true
                                 │         Proceed to test suite
                                 └─ NO  ──► rejectionReason set
                                            Fallback to FullSuite (all tests)

Commit SHA changes → Controller sets phase=Stale
                    Next reconcile creates a new TestPlan
```

## Usage Patterns

### Automatic (AI-driven)

The most common pattern. Set in the Preview:

```yaml
spec:
  testStrategy:
    mode: Auto
    confidenceThreshold: 70       # default
    fallbackOnAgentTimeout: Full  # Skipped if agent times out
    agentTimeoutSeconds: 60
```

The controller automatically creates a TestPlan, calls the agent, and manages the lifecycle.

### Manual (developer-specified)

Override the agent's decision with a hand-crafted TestPlan:

```yaml
apiVersion: platform.company.io/v1alpha1
kind: TestPlan
metadata:
  name: pr-42-manual
  namespace: preview-pr-42
spec:
  generatedBy: Manual
  previewRef:
    kind: Preview
    name: pr-42
  mustRun:
    - suite: smoke
      reason: "Always."
    - suite: e2e
      reason: "Frontend critical for this PR."
  canSkip:
    - suite: contract
      reason: "Backend didn't change."
  confidence: 100  # Manual plans should have high confidence
```

Then reference it in the Preview:

```yaml
spec:
  testStrategy:
    mode: Manual
    manualPlanRef:
      name: pr-42-manual
```

### Fallback (on timeout)

If the agent doesn't respond within `agentTimeoutSeconds`, the controller falls back to a policy-based plan:

```yaml
spec:
  testStrategy:
    mode: Auto
    fallbackOnAgentTimeout: Full  # Run all enabled test suites
    # OR
    fallbackOnAgentTimeout: Smoke # Run only smoke tests
```

## Kubernetes Operations

### Create a TestPlan manually

```bash
kubectl apply -f - <<EOF
apiVersion: platform.company.io/v1alpha1
kind: TestPlan
metadata:
  name: pr-42-manual
  namespace: preview-pr-42
spec:
  generatedBy: Manual
  previewRef:
    kind: Preview
    name: pr-42
  mustRun:
    - suite: smoke
      name: "*"
  confidence: 100
EOF
```

### Check TestPlan status

```bash
# List TestPlans in the namespace
kubectl get testplan -n preview-pr-42

# Watch as it progresses
kubectl get testplan -n preview-pr-42 -w

# View full spec and status
kubectl get testplan pr-42-abc123 -n preview-pr-42 -o jsonpath='{.spec}' | jq .
kubectl get testplan pr-42-abc123 -n preview-pr-42 -o jsonpath='{.status}' | jq .
```

### View the trigger Job

```bash
# Job that calls the agent
kubectl get jobs -n preview-pr-42 | grep trigger
kubectl logs -n preview-pr-42 job/trigger-pr-42-xxxxx

# See the agent's response in TestPlan spec
kubectl get testplan -n preview-pr-42 -o jsonpath='{.items[0].spec}' | jq '{mustRun,canSkip,confidence,rationale}'
```

### Force a plan to expire (triggers new agent call)

```bash
# Mark the current plan Stale to trigger a fresh agent call
kubectl patch testplan pr-42-abc123 -n preview-pr-42 --type=merge \
  -p '{"status":{"phase":"Stale"}}'
```

## Relationships

```
TestPlan (N) ◄─── Preview (1)
  ├─ references → Preview (previewRef)
  │
  └─ references ← TestRun (testPlanRef)
```

- **One Preview** can have multiple TestPlans (one per reconcile cycle or manual override)
- **TestRun** references which TestPlan it followed
- **ReconcileEvent** may record agent success/failure for historical signal

## Agent Integration

The test-strategist-agent is a **kagent Agent** that:

1. **Reads** the Preview CR's `spec.changeContext` (diff files, detectedImpacts, raw diff)
2. **Analyzes** which files changed and what impact they have
3. **Decides** which suites are mandatory, recommended, or safe to skip
4. **Writes** a TestPlan CR with its recommendation + confidence score
5. **Returns** success; controller polls and validates the result

### File Impact Rules

The agent uses file patterns to infer impact:

| File pattern | Classification | Default decision |
|--------------|-----------------|-----------------|
| `db/migrations/` or `alembic/versions/` | database-migration | mustRun: migration, regression, smoke |
| `api/openapi.yaml` or `openapi.json` | api-contract | mustRun: contract, regression, smoke |
| `frontend.py`, `templates/`, `static/`, `web/` | frontend | mustRun: e2e, regression, smoke |
| `app.py`, `src/`, `tests/` | backend | mustRun: regression, smoke |
| `README`, `*.md`, `docs/` | docs | canSkip: all |

### Example: agent output

For a PR that changes only `app.py`:

```yaml
spec:
  mustRun:
    - suite: smoke
      reason: "Smoke always runs — baseline liveness."
    - suite: regression
      reason: "Backend changes require regression coverage."
  canSkip:
    - suite: migration
      reason: "No migration files changed."
    - suite: contract
      reason: "API spec unchanged; contract testing deferred."
    - suite: e2e
      reason: "Frontend untouched; E2E suite can skip."
  confidence: 82
  rationale: "Backend logic changed in app.py. Regression tests cover business logic. E2E and contract deferred."
```

## Troubleshooting

### TestPlan stuck in Pending

```bash
# Check if the trigger Job exists
kubectl get jobs -n preview-pr-42 | grep trigger

# Check Job logs
kubectl logs -n preview-pr-42 job/trigger-pr-42-xxxxx

# Check if agent is ready
kubectl get agent test-strategist-agent -n kagent-system
# Expected: READY=True ACCEPTED=True
```

### Agent timeout (fallback to Full)

```bash
# Check Preview to see fallback was applied
kubectl get preview pr-42 -o jsonpath='{.status.testPlanResolution}'

# Increase timeout for next run
kubectl patch preview pr-42 --type=merge \
  -p '{"spec":{"testStrategy":{"agentTimeoutSeconds":120}}}'
```

### Confidence below threshold (rejection)

```bash
# Check rejection reason
kubectl get testplan pr-42-abc123 -n preview-pr-42 \
  -o jsonpath='{.status.rejectionReason}'

# Check the plan to understand why confidence was low
kubectl get testplan pr-42-abc123 -n preview-pr-42 \
  -o jsonpath='{.spec.rationale}'

# Lower the threshold temporarily to test
kubectl patch preview pr-42 --type=merge \
  -p '{"spec":{"testStrategy":{"confidenceThreshold":50}}}'
```

### Stale plan (commit SHA changed)

```bash
# Mark the plan as stale to trigger a fresh call
kubectl patch testplan pr-42-abc123 -n preview-pr-42 --type=merge \
  -p '{"status":{"phase":"Stale"}}'
```
